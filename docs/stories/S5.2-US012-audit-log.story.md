## Descrição

> Como **administrador ou fiscal da Controladoria**, quero consultar o log de auditoria completo do sistema e o histórico detalhado de tramitação de cada ordem, para que todas as ações realizadas no sistema sejam rastreáveis e auditáveis para prestação de contas.

## Contexto e Motivação

A auditoria é um requisito legal fundamental para sistemas de compras públicas. Cada ação deve deixar um rastro imutável. Esta US expõe ao administrador o log global de ações do sistema (login, logout, mudanças de perfil) e o histórico completo de tramitação de cada ordem.

## Critérios de Aceitação

### Cenário 1: Admin acessa log global de auditoria
```gherkin
Given que estou autenticado com perfil "admin"
When acesso a tela "Audit Log"
Then vejo todos os registros de audit_logs do sistema
And cada registro exibe: data/hora, usuário, ação (LOGIN, LOGOUT, LOGIN_FAILED, etc.), IP, user_agent
And posso filtrar por: usuario_id, ação, período (data_inicio, data_fim), secretaria_id
```

### Cenário 2: Histórico de tramitação de uma ordem no detalhe
```gherkin
Given que estou visualizando o detalhe de uma ordem
When acesso a aba ou seção "Histórico de Tramitação"
Then vejo todos os eventos da ordem em ordem cronológica crescente
And cada evento exibe: data/hora, ação, perfil, nome do usuário, observação, status anterior, status novo
And o histórico é append-only — nenhum registro foi alterado ou removido
```

### Cenário 3: Secretaria vê histórico apenas das próprias ordens
```gherkin
Given que estou autenticado com perfil "secretaria"
When acesso o histórico de uma ordem da minha secretaria
Then vejo o histórico completo dessa ordem
When tento acessar o histórico de uma ordem de outra secretaria via URL
Then recebo HTTP 403: "Acesso negado"
```

### Cenário 4: Controladoria vê histórico de qualquer ordem
```gherkin
Given que estou autenticado com perfil "controladoria"
When acesso o histórico de qualquer ordem do sistema
Then posso visualizar o histórico completo
And não posso acessar o log global de auditoria (somente admin)
```

### Cenário 5: Filtros no log global de auditoria
```gherkin
Given que estou na tela de Audit Log como admin
When aplico filtros por período (ex: última semana) e ação (LOGIN_FAILED)
Then o log exibe somente os registros que correspondem aos filtros
And a paginação respeita os filtros aplicados
```

### Cenário 6: Imutabilidade do log
```gherkin
Given que qualquer ação é executada no sistema
When o evento é registrado em audit_logs ou ordem_historico
Then o registro nunca pode ser alterado ou deletado por nenhum usuário
And qualquer tentativa de DELETE ou UPDATE nessas tabelas é bloqueada pela arquitetura
```

### Cenário 7: Exportação do histórico de uma ordem
```gherkin
Given que estou visualizando o histórico de uma ordem
When clico em "Exportar PDF"
Then o sistema gera um PDF com o histórico completo da ordem
And o PDF contém: dados da ordem, linha do tempo de tramitação, assinaturas dos responsáveis
```

## Regras de Negócio

| ID | Regra |
|----|-------|
| RN-60 | Log de auditoria é append-only — nenhum registro pode ser alterado ou deletado |
| RN-61 | Cada entrada contém: ordem_id, usuario_id, perfil, acao, status_anterior, status_novo, observacao, ip_address, created_at |
| RN-62 | Histórico disponível para: Admin, Controladoria e Secretaria de origem da ordem |
| RN-63 | Exportação do histórico de uma ordem em formato PDF |
| RN-64 | Logs de acesso ao sistema (login/logout) armazenados separadamente em audit_logs |
