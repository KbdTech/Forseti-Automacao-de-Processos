# BUG-001 — Formatação de Campos de Valor Monetário

**ID:** BUG-001
**Sprint:** S7
**Status:** Ready
**Prioridade:** Crítica
**Depende de:** —
**Módulo:** Pipeline Financeiro (Empenho, Liquidação, Pagamento)

---

## Descrição

> Como **usuário do sistema**, quero que os campos de valor monetário sejam exibidos e aceitos no formato correto (R$ 1.234,56), para que eu não precise apagar manualmente a vírgula e os decimais antes de submeter o formulário.

## Contexto e Motivação

Campos de valor (especialmente `valor_empenhado`) são pré-preenchidos com o valor numérico bruto do banco. O campo exibe vírgula e casas decimais sem formatação, obrigando o usuário a apagar caracteres manualmente para conseguir submeter. Isso bloqueia o fluxo financeiro.

---

## Critérios de Aceitação

### Cenário 1: Valor empenhado pré-preenchido com formatação correta
```gherkin
Given que acesso o modal de empenho de uma ordem com valor_estimado de 10000.00
When o modal abre
Then o campo "Valor Empenhado" exibe "R$ 10.000,00"
And posso editar o valor sem precisar apagar vírgula ou decimais manualmente
```

### Cenário 2: Submissão envia valor numérico correto à API
```gherkin
Given que o campo exibe "R$ 9.500,00"
When clico em confirmar
Then a API recebe o payload com valor_empenhado: 9500.00 (número, sem símbolo)
And a ordem é atualizada corretamente
```

### Cenário 3: Campos afetados em todos os modais financeiros
```gherkin
Given que abro qualquer modal financeiro (Empenho, Liquidação, Pagamento)
Then todos os campos de valor monetário usam formatação R$ X.XXX,XX
And nenhum campo exige limpeza manual para avançar o fluxo
```

---

## Especificação Técnica

### Front-End

**Utilitário existente** (`frontend/src/utils/formatters.ts`):
- Usar/criar `formatCurrency(value: number): string` → `"R$ 1.234,56"`
- Usar/criar `parseCurrency(str: string): number` → `1234.56`

**Campos a corrigir:**
- Modal de Empenho: `valor_empenhado` (pré-preenchido com `ordem.valor_estimado`)
- Modal de Liquidação: `valor_liquidado`
- Modal de Pagamento: `valor_pago`

**Padrão para inputs de moeda:**
```typescript
// Ao pré-preencher
setValue('valor_empenhado', formatCurrency(ordem.valor_estimado))

// Ao submeter (antes de enviar à API)
const payload = {
  valor_empenhado: parseCurrency(getValues('valor_empenhado'))
}
```

**Usar componente `CurrencyInput`** (se existente) ou aplicar máscara com `onChangeCapture`.

### Back-End

Sem alterações necessárias — o back-end já recebe e persiste `DECIMAL(15,2)`.
