## Descrição

> Como **administrador**, quero uma tela completa para cadastrar e gerenciar empresas vencedoras de licitação, para que os dados estejam disponíveis no sistema antes que as secretarias comecem a criar ordens vinculadas a fornecedores.

## Contexto e Motivação

Após a API estar disponível (S11.1), o admin precisa de interface para gerenciar o cadastro de fornecedores. A tela segue os padrões visuais estabelecidos no sistema: tabela com filtros, dialogs de criação/edição e AlertDialog de confirmação para ações críticas — igual ao padrão da `SecretariaManagementPage.tsx`.

## Critérios de Aceitação

### Cenário 1: Tabela exibe fornecedores com colunas corretas

```gherkin
Given que acesso /admin/fornecedores como admin
Then vejo uma tabela com colunas:
  - Razão Social
  - CNPJ (formatado XX.XXX.XXX/XXXX-XX)
  - Nº Processo
  - Valor Contratado (R$ X.XXX,XX)
  - Secretaria (ou "Global" se null)
  - Status (badge Ativo/Inativo)
  - Ações (botões Editar, Ativar/Desativar)
And skeleton loaders são exibidos enquanto os dados carregam
```

### Cenário 2: Filtros funcionam corretamente

```gherkin
Given que estou na tela de gestão de fornecedores
When digito "CNPJ" ou "nome" no campo de busca (debounce 300ms)
Then a tabela atualiza mostrando apenas fornecedores correspondentes

When seleciono uma secretaria no filtro de secretaria
Then vejo apenas fornecedores dessa secretaria (ou globais)

When seleciono "Ativos" ou "Inativos" no filtro de status
Then a tabela filtra corretamente
```

### Cenário 3: Cadastrar novo fornecedor

```gherkin
Given que clico em "Novo Fornecedor"
Then abre um Dialog com campos:
  - Razão Social * (obrigatório)
  - Nome Fantasia (opcional)
  - CNPJ * (obrigatório, máscara XX.XXX.XXX/XXXX-XX, validação no submit)
  - Nº Processo Licitatório (opcional)
  - Objeto do Contrato (opcional, textarea)
  - Valor Contratado (opcional, máscara BRL — padrão BUG-001)
  - Data do Contrato (opcional, input date)
  - Banco (opcional)
  - Agência (opcional)
  - Conta (opcional)
  - Tipo de Conta (opcional, select: Corrente / Poupança)
  - Secretaria (opcional, select — null = Global)

When preencho campos obrigatórios e clico "Salvar"
Then o fornecedor é cadastrado
And toast de sucesso: "Fornecedor cadastrado com sucesso."
And o Dialog fecha
And a tabela atualiza exibindo o novo fornecedor
```

### Cenário 4: CNPJ duplicado exibe erro inline

```gherkin
Given que já existe um fornecedor com o CNPJ informado
When clico em "Salvar" no Dialog de criação
Then recebo HTTP 409
And o campo CNPJ exibe mensagem inline: "CNPJ já cadastrado no sistema."
And o Dialog permanece aberto
```

### Cenário 5: Editar fornecedor

```gherkin
Given que clico em "Editar" na linha de um fornecedor
Then abre o Dialog de edição com os dados atuais preenchidos
And o campo CNPJ é exibido mas NÃO é editável (apenas leitura)

When altero dados e clico em "Salvar"
Then recebo HTTP 200
And toast de sucesso: "Fornecedor atualizado com sucesso."
And a tabela atualiza com os novos dados
```

### Cenário 6: Desativar fornecedor

```gherkin
Given que clico em "Desativar" na linha de um fornecedor ativo
Then abre AlertDialog: "Deseja desativar este fornecedor? Ele não poderá ser selecionado em novas ordens."
When clico em "Confirmar"
Then o fornecedor fica inativo
And o badge muda para "Inativo"
And toast de sucesso: "Fornecedor desativado."
```

### Cenário 7: Ativar fornecedor inativo

```gherkin
Given que clico em "Ativar" na linha de um fornecedor inativo
Then abre AlertDialog de confirmação
When confirmo
Then o fornecedor fica ativo novamente
And toast de sucesso: "Fornecedor ativado."
```

### Cenário 8: Empty state

```gherkin
Given que não há fornecedores cadastrados (ou os filtros retornam vazio)
Then exibo ícone + mensagem "Nenhum fornecedor encontrado."
And se não há filtros ativos: botão "Cadastrar Primeiro Fornecedor"
```

### Cenário 9: Rota protegida para admin

```gherkin
Given que acesso /admin/fornecedores com qualquer perfil que não seja "admin"
Then sou redirecionado para /acesso-negado
```

### Formatação CNPJ

```typescript
// utils/formatters.ts — adicionar
export function formatCNPJ(cnpj: string): string {
  const digits = cnpj.replace(/\D/g, '')
  return digits.replace(/(\d{2})(\d{3})(\d{3})(\d{4})(\d{2})/, '$1.$2.$3/$4-$5')
}

export function parseCNPJ(formatted: string): string {
  return formatted.replace(/\D/g, '')
}
```

### Valor Contratado — padrão BUG-001

Campo `valor_contratado` usa `type="text"` com `parseBRL`/`formatCurrencyInput` importados de `utils/formatters.ts` (mesmo padrão de EmpenhoModal, PagamentoModal etc.).
