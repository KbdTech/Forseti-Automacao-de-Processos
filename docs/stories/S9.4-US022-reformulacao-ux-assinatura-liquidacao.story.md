# US-022 — Reformulação UX do Fluxo de Assinatura na Liquidação

**ID:** US-022
**Sprint:** S9
**Status:** Done
**Prioridade:** Alta
**Depende de:** US-019 (S8 — status e página já implementados)
**Módulo:** Frontend / AssinaturaLiquidacaoPage

---

## Descrição

> Como **secretaria**, ao clicar em "Assinar" na fila de liquidações, quero ver um modal completo com os dados da ordem, listagem de documentos para download e campo para upload do documento assinado — em vez de apenas uma confirmação simples — para assinar com consciência e registrar o comprovante no sistema enquanto a integração com Clicksign não está disponível.

## Contexto e Motivação

A US-019 (Sprint 8) implementou o status `AGUARDANDO_ASSINATURA_SECRETARIA` e a página `AssinaturaLiquidacaoPage.tsx`, que contém o componente interno `AssinaturaConfirmModal`. O modal atual é uma **confirmação simples**: mostra protocolo, secretaria, valor liquidado, data e apenas o botão "Confirmar Assinatura" — sem documentos, sem upload, sem nenhuma evidência da assinatura registrada.

O processo real exige que o secretário confira os documentos e registre o arquivo assinado como evidência no sistema.

## Critérios de Aceitação

### Cenário 1: Modal exibe dados e listagem de documentos
```gherkin
Given que clico em "Assinar" em uma ordem em "AGUARDANDO_ASSINATURA_SECRETARIA"
Then abre o modal com:
  - Protocolo, Secretaria, Valor Liquidado, Data da Liquidação
  - Listagem de todos os documentos da ordem com botão de download
```

### Cenário 2: Upload de documento assinado é obrigatório
```gherkin
Given que o modal está aberto
When clico em "Confirmar Assinatura" sem anexar nenhum arquivo
Then o botão permanece desabilitado
And uma mensagem indica que o documento assinado é obrigatório
```

### Cenário 3: Checkbox de confirmação é obrigatório
```gherkin
Given que o modal está aberto e um arquivo foi anexado
But o checkbox de confirmação não está marcado
Then o botão "Confirmar Assinatura" permanece desabilitado
```

### Cenário 4: Confirmação com sucesso
```gherkin
Given que o modal está aberto
And um arquivo foi anexado
And o checkbox está marcado
When clico em "Confirmar Assinatura"
Then o documento é salvo com descricao="ASSINATURA_LIQUIDACAO"
And a ação "assinar_liquidacao" é executada
And a ordem avança para "AGUARDANDO_PAGAMENTO"
And o modal fecha automaticamente
And toast de sucesso é exibido
```

### Cenário 5: Tesouraria só vê a ordem após confirmação
```gherkin
Given que uma ordem está em "AGUARDANDO_ASSINATURA_SECRETARIA"
Then a ordem NÃO aparece no Pipeline de Pagamento da Tesouraria
When a secretaria confirma a assinatura
Then a ordem aparece no Pipeline de Pagamento
```

**Novo `onConfirm` assíncrono (substituir a chamada direta de `mutation.mutate`):**
```tsx
// No componente pai (AssinaturaLiquidacaoPage), a mutation deve:
// 1. Fazer upload do documento assinado
// 2. Chamar executeAcao('assinar_liquidacao')
mutationFn: async (orderId: string) => {
  if (documentoAssinado) {
    await uploadDocumento(orderId, {
      file: documentoAssinado,
      descricao: 'ASSINATURA_LIQUIDACAO',
    })
  }
  return executeAcao(orderId, { acao: 'assinar_liquidacao' })
}
```

**Seção de documentos dentro do modal** (reutilizar `DocumentList`):
```tsx
<div className="space-y-2">

    ordemId={ordem.id}
    currentUserId={undefined}
    currentUserRole="secretaria"
    readOnly  // sem botão de exclusão
  />
</div>
```

**Reset de estado ao fechar o modal:**
```tsx
function handleClose() {
  setDocumentoAssinado(null)
  setConfirmacaoLida(false)
  if (fileInputRef.current) fileInputRef.current.value = ''
  onClose()
}
```
