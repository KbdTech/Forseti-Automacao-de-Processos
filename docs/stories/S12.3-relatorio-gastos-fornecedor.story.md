## Descrição

> Como **gestora municipal** (gabinete, controladoria, tesouraria, contabilidade) ou **secretaria**, quero visualizar um relatório consolidado de gastos por fornecedor no período, com gráfico mensal e exportação CSV, para acompanhar e auditar o destino dos pagamentos municipais realizados via sistema.

## Contexto e Motivação

Com os fornecedores vinculados às ordens (S11.3), é possível calcular os gastos reais por empresa a partir das ordens com status `PAGA`. O gabinete precisa dessa visão no dashboard executivo; os demais perfis financeiros precisam de uma página de relatório dedicada com filtros por período e secretaria.

## Especificação Técnica

### Backend — Novo Endpoint

**Arquivo:** `backend/app/api/routes/dashboard.py`

```python
@router.get("/gastos-fornecedor")
async def gastos_por_fornecedor(
    current_user: AnyAuthenticated,
    db: Annotated[AsyncSession, Depends(get_db)],
    data_inicio: date | None = Query(None),
    data_fim: date | None = Query(None),
    secretaria_id: uuid.UUID | None = Query(None),
) -> list[GastoFornecedorResponse]:
    """
    Retorna gastos consolidados por fornecedor.
    Scoping: secretaria vê apenas própria secretaria.
    Perfis globais: veem tudo ou filtram por secretaria_id.
    """
```

**Schema de resposta:**

```python
class GastoFornecedorResponse(BaseModel):
    fornecedor_id: uuid.UUID
    razao_social: str
    cnpj: str
    total_pago: Decimal
    count_ordens: int
    secretaria_nome: str | None
```

**Query SQLAlchemy:**

```python
stmt = (
    select(
        Ordem.fornecedor_id,
        Fornecedor.razao_social,
        Fornecedor.cnpj,
        func.sum(Ordem.valor_pago).label("total_pago"),
        func.count(Ordem.id).label("count_ordens"),
        Secretaria.nome.label("secretaria_nome"),
    )
    .join(Fornecedor, Ordem.fornecedor_id == Fornecedor.id)
    .join(Secretaria, Ordem.secretaria_id == Secretaria.id, isouter=True)
    .where(Ordem.status == StatusOrdemEnum.PAGA)
    .where(Ordem.fornecedor_id.is_not(None))
)

# Filtro de período
if data_inicio_dt:
    stmt = stmt.where(Ordem.updated_at >= data_inicio_dt)
if data_fim_dt:
    stmt = stmt.where(Ordem.updated_at <= data_fim_dt)

# Scoping por secretaria
if current_user.role == RoleEnum.secretaria:
    stmt = stmt.where(Ordem.secretaria_id == current_user.secretaria_id)
elif secretaria_id:
    stmt = stmt.where(Ordem.secretaria_id == secretaria_id)

stmt = (
    stmt
    .group_by(Ordem.fornecedor_id, Fornecedor.razao_social, Fornecedor.cnpj, Secretaria.nome)
    .order_by(func.sum(Ordem.valor_pago).desc())
)
```

> **Nota:** `updated_at` nas ordens PAGAS corresponde ao momento do pagamento — usar como proxy de data de pagamento. Alternativamente, usar `data_pagamento` se o campo estiver no modelo.

```tsx
export default function GastosFornecedorPage() {
  const [dataInicio, setDataInicio] = useState(primeiroDiaDoMes())
  const [dataFim, setDataFim]       = useState(hoje())
  const [secretariaFilter, setSecretariaFilter] = useState<string>('TODAS')

  const { data, isLoading, isError } = useQuery({
    queryKey: ['gastos-fornecedor', dataInicio, dataFim, secretariaFilter],
    queryFn: () => getGastosFornecedor({
      data_inicio: dataInicio,
      data_fim: dataFim,
      secretaria_id: secretariaFilter !== 'TODAS' ? secretariaFilter : undefined,
    }),
  })

  function handleExportarCSV() {
    if (!data?.length) return
    const csv = [
      'Razão Social,CNPJ,Total Pago,Nº Ordens,Secretaria',
      ...data.map(r =>
        `"${r.razao_social}","${r.cnpj}",${r.total_pago},${r.count_ordens},"${r.secretaria_nome ?? 'Global'}"`
      )
    ].join('\n')
    const blob = new Blob([csv], { type: 'text/csv;charset=utf-8;' })
    const url = URL.createObjectURL(blob)
    const link = document.createElement('a')
    link.href = url
    link.download = `gastos-fornecedor-${dataInicio}-${dataFim}.csv`
    link.click()
    URL.revokeObjectURL(url)
  }
  ...
}
```
