# US-019 — Assinatura da Secretaria na Liquidação

**ID:** US-019
**Sprint:** S8
**Status:** Draft
**Prioridade:** Alta
**Depende de:** US-010, US-015, US-018 (S7 concluída)
**Módulo:** Pipeline Operacional (Financeiro)

---

## Descrição

> Como **sistema**, quero que após a contabilidade registrar a liquidação com o documento, o processo retorne à secretaria para o secretário assinar e aprovar antes de ir à tesouraria, garantindo dupla validação no ciclo de liquidação.

## Contexto e Motivação

O documento de liquidação é um ato formal que precisa da assinatura do secretário responsável antes do pagamento ser autorizado. O fluxo atual envia direto da liquidação para o pagamento, sem essa validação. É necessário introduzir um novo status intermediário: `AGUARDANDO_ASSINATURA_SECRETARIA`.

---

## Novo Status na Máquina de Estados

```
AGUARDANDO_LIQUIDACAO (contabilidade)
  └─► AGUARDANDO_ASSINATURA_SECRETARIA  ← NOVO
        └─► AGUARDANDO_PAGAMENTO (tesouraria)
```

**Tabela de transições novas:**

| Status Atual | Ação | Perfil | Próximo Status |
|---|---|---|---|
| AGUARDANDO_LIQUIDACAO | liquidar | contabilidade | AGUARDANDO_ASSINATURA_SECRETARIA |
| AGUARDANDO_ASSINATURA_SECRETARIA | assinar_liquidacao | secretaria | AGUARDANDO_PAGAMENTO |

---

## Critérios de Aceitação

### Cenário 1: Contabilidade faz upload e registra liquidação
```gherkin
Given que estou autenticada com perfil "contabilidade"
And acesso Pipeline Liquidação com ordem em "AGUARDANDO_LIQUIDACAO"
When abro o modal de liquidação
Then vejo: valor liquidado, data de liquidação e campo de upload de documento (obrigatório)
And não há campo de observação
When preencho os dados e anexo o documento e clico em confirmar
Then a ordem avança para "AGUARDANDO_ASSINATURA_SECRETARIA"
And documento salvo com tipo "LIQUIDACAO"
And modal fecha automaticamente (UX-001)
```

### Cenário 2: Secretaria vê pipeline "Aguardando Minha Assinatura"
```gherkin
Given que estou autenticada com perfil "secretaria"
When acesso o menu lateral
Then vejo a opção "Aguardando Minha Assinatura" (ou "Liquidações para Assinar")
And vejo ordens da minha secretaria com status "AGUARDANDO_ASSINATURA_SECRETARIA"
```

### Cenário 3: Secretaria assina e aprova liquidação
```gherkin
Given que acesso uma ordem em "AGUARDANDO_ASSINATURA_SECRETARIA"
When clico em "Assinar e Aprovar"
And confirmo no modal de confirmação
Then a ordem avança diretamente para "AGUARDANDO_PAGAMENTO"
And um registro em ordem_historico: acao="assinar_liquidacao", perfil="secretaria"
And modal fecha automaticamente (UX-001)
```

### Cenário 4: Secretaria não pode alterar documentos nesta etapa
```gherkin
Given que acesso uma ordem em "AGUARDANDO_ASSINATURA_SECRETARIA"
Then posso apenas visualizar o documento de liquidação (download)
And não há campo de upload nem edição disponível
```

### Cenário 5: Tesouraria só vê a ordem após assinatura da secretaria
```gherkin
Given que uma ordem está em "AGUARDANDO_ASSINATURA_SECRETARIA"
When acesso o Pipeline Pagamento como tesouraria
Then a ordem NÃO aparece no pipeline
When a secretaria assina e a ordem vai para "AGUARDANDO_PAGAMENTO"
Then a ordem aparece no Pipeline Pagamento
```

## Especificação Técnica

### Back-End

**Migration Alembic (009):**
```sql
-- Adicionar novo valor ao ENUM status_ordem
ALTER TYPE status_ordem ADD VALUE 'AGUARDANDO_ASSINATURA_SECRETARIA';
```

**Workflow engine** — novas transições:
```python
# ordem_service.py
TRANSITIONS = {
  ...
  'AGUARDANDO_LIQUIDACAO': {
    'liquidar': ('AGUARDANDO_ASSINATURA_SECRETARIA', 'contabilidade'),
  },
  'AGUARDANDO_ASSINATURA_SECRETARIA': {
    'assinar_liquidacao': ('AGUARDANDO_PAGAMENTO', 'secretaria'),
  },
}
```

**Payload ação `liquidar` (atualizado):**
```json
{
  "acao": "liquidar",
  "valor_liquidado": 0.00,
  "data_liquidacao": "YYYY-MM-DD"
  // sem observacao
  // documento enviado via POST /documentos antes desta chamada
}
```

**Novo payload ação `assinar_liquidacao`:**
```json
{ "acao": "assinar_liquidacao" }
```

**Filtro de ordens por perfil** — atualizar para incluir:
- `secretaria`: recebe também `AGUARDANDO_ASSINATURA_SECRETARIA` (filtrado por secretaria_id)

### Front-End

**Novas páginas/componentes:**
- Novo item no menu lateral para secretaria: *"Liquidações para Assinar"*
- Nova página/seção listando ordens em `AGUARDANDO_ASSINATURA_SECRETARIA`
- Modal simples de confirmação de assinatura (sem campos adicionais)
