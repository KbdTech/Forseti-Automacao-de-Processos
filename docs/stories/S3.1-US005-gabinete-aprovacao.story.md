# US-005 — Análise e Decisão do Gabinete do Prefeito

**ID:** US-005
**Sprint:** S3 (Semanas 5–6)
**Status:** Ready
**Prioridade:** Alta
**Depende de:** US-003, US-004
**Estimativa:** 13 pontos
**Módulo:** Workflow de Aprovação

---

## Descrição

> Como **membro da equipe do Gabinete do Prefeito**, quero analisar as ordens aguardando aprovação e tomar decisões de autorizar, solicitar alterações ou cancelar, para que apenas demandas legítimas e bem justificadas avancem no processo de compras públicas.

## Contexto e Motivação

O Gabinete é o primeiro filtro de qualidade do fluxo. Toda ordem criada passa por aqui antes de seguir para a Controladoria. Esta US implementa o endpoint de ação de workflow e conecta a tela do Gabinete às transições reais de status no banco.

## Critérios de Aceitação

### Cenário 1: Visualização da fila de ordens pendentes
```gherkin
Given que estou autenticado com perfil "gabinete"
When acesso a tela "Pipeline Gabinete"
Then vejo todas as ordens com status "AGUARDANDO_GABINETE" do sistema
And cada linha exibe: protocolo, secretaria, tipo, prioridade, data criação, valor estimado
And a lista está ordenada por data de criação (mais antigas primeiro)
```

### Cenário 2: Autorizar uma ordem
```gherkin
Given que estou na tela do Gabinete e seleciono uma ordem com status "AGUARDANDO_GABINETE"
When clico em "Autorizar" e confirmo no modal
Then a ordem é atualizada para status "AGUARDANDO_CONTROLADORIA"
And um registro é inserido em ordem_historico: acao="autorizar", perfil="gabinete", status_anterior, status_novo
And a secretaria recebe notificação (assíncrona, US-014)
And um toast de sucesso é exibido: "Ordem OS-XXXX autorizada com sucesso"
```

### Cenário 3: Solicitar alterações com observação obrigatória
```gherkin
Given que estou analisando uma ordem com status "AGUARDANDO_GABINETE"
When clico em "Solicitar Alterações"
And preencho a observação com pelo menos 20 caracteres
And confirmo no modal
Then a ordem muda para status "DEVOLVIDA_PARA_ALTERACAO"
And a observação é registrada em ordem_historico
And a secretaria é notificada (assíncrona)
```

### Cenário 4: Solicitar alterações sem observação — bloqueado
```gherkin
Given que clico em "Solicitar Alterações"
When submeto o formulário sem preencher a observação ou com menos de 20 caracteres
Then o campo exibe erro: "Observação é obrigatória e deve ter no mínimo 20 caracteres"
And nenhuma requisição é enviada ao back-end
```

### Cenário 5: Cancelar uma ordem
```gherkin
Given que estou analisando uma ordem com status "AGUARDANDO_GABINETE"
When clico em "Cancelar Ordem"
And preencho o motivo de cancelamento
And confirmo no modal com aviso de que a ação é IRREVERSÍVEL
Then a ordem muda para status "CANCELADA" (terminal)
And o registro em ordem_historico reflete o cancelamento
And a ordem não pode mais receber nenhuma ação
```

### Cenário 6: Ação em ordem com status incorreto é rejeitada
```gherkin
Given que uma ordem está com status "AGUARDANDO_CONTROLADORIA"
When o back-end recebe uma requisição de ação "autorizar" do Gabinete para essa ordem
Then recebo HTTP 422: "Transição inválida: ordem não está no status AGUARDANDO_GABINETE"
And nenhuma alteração é feita no banco
```

### Cenário 7: Gabinete visualiza qualquer ordem em modo somente-leitura
```gherkin
Given que estou autenticado com perfil "gabinete"
When acesso o detalhe de uma ordem com status "PAGA" ou "AGUARDANDO_CONTROLADORIA"
Then posso visualizar todos os dados e histórico
And os botões de ação (autorizar, solicitar, cancelar) estão ocultos (somente ordens AGUARDANDO_GABINETE os exibem)
```

## Regras de Negócio

| ID | Regra |
|----|-------|
| RN-26 | Somente ordens com status AGUARDANDO_GABINETE podem receber ações desta tela |
| RN-27 | Ao solicitar alterações, o campo observacao é obrigatório (mínimo 20 caracteres) |
| RN-28 | Ao cancelar, o campo motivo do cancelamento é obrigatório |
| RN-29 | Ação de cancelamento é IRREVERSÍVEL — somente o Administrador pode reverter |
| RN-30 | Todo histórico de ação deve registrar: usuário, perfil, data/hora e observação |
| RN-31 | O Gabinete pode visualizar qualquer ordem do sistema em modo somente-leitura |

## Escopo

### IN ✅
- Tela de fila do Gabinete com ordens AGUARDANDO_GABINETE
- Endpoint PATCH /api/ordens/:id/acao para ações: autorizar, solicitar_alteracao, cancelar
- Máquina de estados validando transições no back-end
- Modal de observação para solicitar_alteracao (min 20 chars)
- Modal de confirmação com aviso de irreversibilidade para cancelar
- Registro em ordem_historico após cada ação
- Toast de feedback de sucesso/erro

### OUT ❌
- Edição dos dados da ordem pelo Gabinete
- Ações em ordens de outros status
- Desfazer cancelamento (somente Admin, fora desta US)
