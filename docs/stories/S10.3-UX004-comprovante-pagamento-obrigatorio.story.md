# UX-004 — Comprovante de Pagamento Obrigatório

**ID:** UX-004
**Sprint:** S10
**Status:** Draft
**Prioridade:** Alta
**Estimativa:** 3 pts
**Depende de:** US-020 (S7 — comprovante já implementado como opcional)
**Módulo:** Frontend / PagamentoModal + Backend / ordem_service

---

## Descrição

> Como **sistema**, quero que o Comprovante de Pagamento seja obrigatório no registro de pagamento, garantindo que toda ordem paga tenha evidência documental do pagamento para fins de auditoria.

## Contexto e Motivação

A US-020 (Sprint 7) implementou o upload de comprovante de pagamento como **opcional**, com label "Opcional — recomendado para fins de auditoria". O cliente solicita que seja **obrigatório**: nenhum pagamento pode ser registrado sem o comprovante. A estrutura já existe no `PagamentoModal.tsx` — a mudança é adicionar a guarda de habilitação e validação backend.

---

## Especificação Técnica

### Front-End

**Arquivo:** `frontend/src/components/orders/PagamentoModal.tsx`

1. Atualizar guarda `isValid`:
```tsx
// ANTES
const isValid =
  valorNum > 0 &&
  dataPagamento.length > 0 &&
  formaPagamento !== '' &&
  (!obsRequired || observacao.trim().length > 0)

// DEPOIS
const isValid =
  valorNum > 0 &&
  dataPagamento.length > 0 &&
  formaPagamento !== '' &&
  (!obsRequired || observacao.trim().length > 0) &&
  comprovante !== null  // ← NOVO
```

2. Atualizar label do comprovante:
```tsx

<Label>Comprovante de Pagamento</Label>

// DEPOIS
<Label>
  Comprovante de Pagamento <span className="text-destructive">*</span>
</Label>
<p className="text-xs text-muted-foreground">
  Obrigatório para registro de pagamento.
</p>
```

3. Mensagem de orientação quando sem comprovante:
```tsx
{comprovante === null && (
  <p className="text-xs text-destructive mt-1">

```

### Back-End

**Arquivo:** `backend/app/services/ordem_service.py`

Adicionar validação antes de processar a ação `pagar`:
```python
async def _validar_comprovante_pagamento(
    self, ordem_id: UUID, db: AsyncSession
) -> None:
    """Garante que existe comprovante de pagamento antes de registrar o pagamento."""
    result = await db.execute(
        select(Documento).where(
            Documento.ordem_id == ordem_id,
            Documento.descricao == 'COMPROVANTE_PAGAMENTO'
        )
    )
    if not result.scalar_one_or_none():
        raise HTTPException(
            status_code=422,
            detail="Comprovante de pagamento obrigatório"
        )
```

Chamar no handler da ação `pagar`, antes da transição de status.
