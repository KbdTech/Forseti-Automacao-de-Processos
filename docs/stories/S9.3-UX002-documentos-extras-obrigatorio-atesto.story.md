# UX-002 — Documentos Extras obrigatório no Atesto

**ID:** UX-002
**Sprint:** S9
**Status:** Done
**Prioridade:** Alta
**Depende de:** US-018 (S7 — campo já existe como opcional em AtesteModal)
**Módulo:** Frontend / AtesteModal | Backend / ordem_service

---

## Descrição

> Como **sistema**, quero que o campo "Documentos Extras" no modal de Atesto seja obrigatório, garantindo que a secretaria sempre anexe a documentação complementar necessária antes de atestar a nota fiscal.

## Contexto e Motivação

Na US-018 (Sprint 7), o campo Documentos Extras foi implementado como opcional em `AtesteModal.tsx`. O label atual exibe **"Documentos Extras (opcional)"** com visual diferente do campo de Nota Fiscal obrigatório. Tornando-o obrigatório, o visual deve ser idêntico ao campo **"Arquivo da Nota Fiscal *"**: mesma área de drop/clique com borda tracejada, ícone de clipe, asterisco vermelho no label e mensagem de orientação.

---

### Back-End

**Arquivo:** `backend/app/services/ordem_service.py`

Adicionar validação antes de processar a ação `atestar`. Usar `descricao='DOCUMENTO_ATESTO'` — string exata usada pelo frontend:

```python
async def _validar_documentos_atesto(
    self, ordem_id: UUID, db: AsyncSession
) -> None:
    """Garante ao menos um documento extra (DOCUMENTO_ATESTO) para o atesto."""
    result = await db.execute(
        select(Documento).where(
            Documento.ordem_id == ordem_id,
            Documento.descricao == 'DOCUMENTO_ATESTO'
        )
    )
    if not result.scalar_one_or_none():
        raise HTTPException(
            status_code=422,
            detail="Documento extra obrigatório para o atesto"
        )
```

Chamar dentro do handler da ação `atestar`, **antes** da transição de status.
