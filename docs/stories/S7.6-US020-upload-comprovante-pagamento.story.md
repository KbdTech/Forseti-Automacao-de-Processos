# US-020 — Upload de Comprovante de Pagamento (Tesouraria)

**ID:** US-020
**Sprint:** S7
**Status:** Ready
**Prioridade:** Média
**Depende de:** US-010, US-015
**Módulo:** Pipeline Operacional (Financeiro)

---

## Descrição

> Como **tesoureira**, quero poder anexar o comprovante de pagamento ao registrar o pagamento de uma ordem, para que haja evidência documental do pagamento realizado vinculada à ordem no sistema.

## Contexto e Motivação

O comprovante de pagamento (transferência bancária, PIX, cheque) é o documento final que encerra o ciclo financeiro de uma ordem. Embora não seja obrigatório para registrar o pagamento, é altamente recomendado que sempre esteja presente para fins de prestação de contas e auditoria.

---

## Critérios de Aceitação

### Cenário 1: Modal de pagamento exibe campo de upload opcional
```gherkin
Given que estou autenticada com perfil "tesouraria"
And acesso o Pipeline Pagamento com uma ordem em "AGUARDANDO_PAGAMENTO"
When abro o modal de pagamento
Then vejo os campos existentes (valor, data, forma de pagamento)
And vejo um campo opcional de upload com label "Comprovante de Pagamento (recomendado)"
And uma mensagem de recomendação é exibida abaixo do campo
```

### Cenário 2: Registrar pagamento com comprovante anexado
```gherkin
Given que preenchi os dados de pagamento e anexei o comprovante
When clico em "Registrar Pagamento" e confirmo
Then a ordem avança para status "PAGA"
And o comprovante é salvo com tipo "COMPROVANTE_PAGAMENTO" vinculado à ordem
And modal fecha automaticamente (UX-001)
```

### Cenário 3: Registrar pagamento sem comprovante (permitido)
```gherkin
Given que preenchi os dados de pagamento mas não anexei comprovante
When clico em "Registrar Pagamento" e confirmo
Then a ordem avança para status "PAGA" normalmente
And nenhum documento de comprovante é criado
And modal fecha automaticamente (UX-001)
```

### Cenário 4: Mensagem de recomendação visível
```gherkin
Given que o campo de comprovante está vazio
Then exibo a mensagem: "Recomendado: anexe o comprovante de pagamento para fins de auditoria"
And a mensagem não bloqueia o envio
```
