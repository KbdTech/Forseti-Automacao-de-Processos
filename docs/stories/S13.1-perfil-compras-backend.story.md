## Descrição

> Como **administrador do sistema**, quero cadastrar usuários com o perfil `compras` para que o Setor de Compras/Licitações possa gerenciar fornecedores de forma autônoma, sem acessar pipelines de ordens ou dados financeiros.

## Contexto e Motivação

O módulo de fornecedores (S11/S12) foi implementado com CRUD exclusivo para `admin`. A prefeitura possui um Setor de Compras/Licitações responsável pelos contratos e cadastro de empresas vencedoras — setor distinto da Contabilidade. Criar o perfil `compras` separa claramente a responsabilidade de cadastrar fornecedores da responsabilidade de empenhar ordens.

O perfil `compras`:
- Acessa **apenas** as rotas de gestão de fornecedores (`GET/POST/PUT/PATCH /api/fornecedores`)
- **Não participa** de nenhuma transição de workflow de ordens
- **Não vê** pipelines, dashboard ou gestão de usuários/secretarias

## Especificação Técnica

### 1. `backend/app/models/user.py` — Adicionar valor ao enum

```python
class RoleEnum(str, enum.Enum):
    secretaria = "secretaria"
    gabinete = "gabinete"
    controladoria = "controladoria"
    contabilidade = "contabilidade"
    tesouraria = "tesouraria"
    compras = "compras"   # ← NOVO S13.1
    admin = "admin"
```

### 2. `backend/alembic/versions/011_compras_role.py` — Migration

```python
"""S13.1: adiciona valor 'compras' ao enum role_enum."""

revision = "011_compras_role"
down_revision = "010_fornecedores"

def upgrade() -> None:
    op.execute("ALTER TYPE role_enum ADD VALUE IF NOT EXISTS 'compras'")

def downgrade() -> None:
    # PostgreSQL não suporta remoção de valor de enum nativamente.
    # Downgrade requer recriação do tipo — deixar como no-op documentado.
    pass
```

> **Gotcha:** `ALTER TYPE ... ADD VALUE` não funciona dentro de transações no PostgreSQL.
> Alembic usa DDL não-transacional para isso. Usar `IF NOT EXISTS` para idempotência.
> A migration deve ter `transactional_ddl = False` se necessário (ver padrão de 009).

### 3. `backend/app/api/routes/fornecedores.py` — Atualizar dependência de Admin

```python
# Antes
AdminRequired = Annotated[User, Depends(require_role(RoleEnum.admin))]

# Depois — compras e admin podem criar/editar/desativar fornecedores
ComprasOrAdmin = Annotated[
    User, Depends(require_role(RoleEnum.compras, RoleEnum.admin))
]
```

Substituir `AdminRequired` por `ComprasOrAdmin` nas seguintes rotas:
- `POST /` (create_fornecedor)
- `PUT /{id}` (update_fornecedor)
- `PATCH /{id}/status` (toggle_status_fornecedor)

`GET /` e `GET /{id}` permanecem com `AnyAuthenticated`.
