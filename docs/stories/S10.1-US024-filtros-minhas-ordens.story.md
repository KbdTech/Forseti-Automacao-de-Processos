# US-024 — Filtros em Minhas Ordens: Período e Prioridade

**ID:** US-024
**Sprint:** S10
**Status:** Draft
**Prioridade:** Alta
**Estimativa:** 8 pts
**Depende de:** US-004 (Acompanhamento de Ordens)
**Módulo:** Frontend / MinhasOrdensPage + Backend / ordens route + service

---

## Descrição

> Como **usuário com perfil secretaria**, quero filtrar as ordens em "Minhas Ordens" por período de criação e por prioridade, para encontrar rapidamente ordens específicas sem precisar percorrer toda a lista paginada.

## Contexto e Motivação

A tela "Minhas Ordens" atualmente suporta busca por protocolo com debounce, mas não possui filtros por data nem por prioridade. Com o crescimento do volume de ordens, esses filtros são essenciais para operação diária. O backend GET /api/ordens já aceita `data_inicio` e `data_fim`, mas **não aceita `prioridade`** — esse param precisa ser adicionado.

---

## Critérios de Aceitação

### Cenário 1: Filtro por prioridade
```gherkin
Given que acesso "Minhas Ordens" como secretaria
When seleciono "Alta" no filtro de prioridade
Then a tabela exibe apenas ordens com prioridade "Alta"
And a paginação reseta para a página 1
And ao limpar o filtro, todas as ordens voltam a aparecer
```

### Cenário 2: Filtro por período de criação
```gherkin
Given que acesso "Minhas Ordens"
When preencho "Data início" = 01/02/2026 e "Data fim" = 28/02/2026
Then a tabela exibe apenas ordens criadas no período informado
And um único campo vazio não aplica o filtro de data
```

### Cenário 3: Filtros combinados
```gherkin
Given que aplico prioridade "Urgente" + data início 01/03/2026
Then a tabela exibe apenas ordens urgentes criadas a partir de 01/03/2026
```

### Cenário 4: Botão "Limpar Filtros"
```gherkin
Given que há filtros aplicados (prioridade e/ou data)
When clico em "Limpar Filtros"
Then todos os filtros são zerados e a lista retorna ao estado inicial
```

### Cenário 5: Backend rejeita enum inválido
```gherkin
Given que envio GET /api/ordens?prioridade=INVALIDA
Then o backend retorna HTTP 422
And a mensagem detalha o valor inválido
```

## Especificação Técnica

### Back-End

**Arquivo:** `backend/app/api/routes/ordens.py`

Adicionar query param no endpoint GET /ordens:
```python
prioridade: Optional[str] = Query(None, description="Filtro por prioridade: NORMAL, ALTA, URGENTE")
```

Validar enum (antes de passar ao service):
```python
if prioridade and prioridade not in ['NORMAL', 'ALTA', 'URGENTE']:
    raise HTTPException(status_code=422, detail=f"Prioridade inválida: {prioridade}")
```

**Arquivo:** `backend/app/services/ordem_service.py`

No método `list_ordens`, adicionar filtro condicional:
```python
if prioridade:
    query = query.where(Ordem.prioridade == prioridade)
```

### Front-End

**Arquivo:** `frontend/src/pages/secretaria/MinhasOrdensPage.tsx`

Novos estados:
```tsx
const [filtroPrioridade, setFiltroPrioridade] = useState<string>('')
const [dataInicio, setDataInicio] = useState<string>('')
const [dataFim, setDataFim] = useState<string>('')
```

Adicionar seção de filtros abaixo da busca por protocolo:
```tsx
<div className="flex flex-wrap gap-3 items-end">
  {/* Filtro prioridade */}
  <Select value={filtroPrioridade} onValueChange={(v) => { setFiltroPrioridade(v); setPage(1) }}>
    <SelectTrigger className="w-[160px]">
      <SelectValue placeholder="Prioridade" />
    </SelectTrigger>
    <SelectContent>
      <SelectItem value="">Todas</SelectItem>
      <SelectItem value="NORMAL">Normal</SelectItem>
      <SelectItem value="ALTA">Alta</SelectItem>
      <SelectItem value="URGENTE">Urgente</SelectItem>
    </SelectContent>
  </Select>

  {/* Data início */}
  <Input
    type="date"
    value={dataInicio}
    onChange={(e) => { setDataInicio(e.target.value); setPage(1) }}
    className="w-[160px]"
    placeholder="Data início"
  />

  {/* Data fim */}
  <Input
    type="date"
    value={dataFim}
    onChange={(e) => { setDataFim(e.target.value); setPage(1) }}
    className="w-[160px]"
    placeholder="Data fim"
  />

  {/* Limpar filtros */}
  {(filtroPrioridade || dataInicio || dataFim) && (
    <Button variant="ghost" size="sm" onClick={handleLimparFiltros}>
      <X className="h-4 w-4 mr-1" />
      Limpar Filtros
    </Button>
  )}
</div>
```

Passar novos params para a query (junto com `protocolo`, `page`, `limit`):
```tsx
queryKey: ['ordens', 'minhas', protocolo, filtroPrioridade, dataInicio, dataFim, page],
queryFn: () => getOrdens({
  protocolo: debouncedProtocolo || undefined,
  prioridade: filtroPrioridade || undefined,
  data_inicio: dataInicio || undefined,
  data_fim: dataFim || undefined,
  page,
  limit: 20,
})
```

Nota: `dataInicio`/`dataFim` não precisam de debounce (inputs de data têm latência natural do picker). `filtroPrioridade` é imediato (select).
