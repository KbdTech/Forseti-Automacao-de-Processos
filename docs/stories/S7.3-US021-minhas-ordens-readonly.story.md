# US-021 — Minhas Ordens: Modo Somente Leitura

**ID:** US-021
**Sprint:** S7
**Status:** Ready
**Prioridade:** Alta
**Depende de:** US-003, US-004
**Módulo:** Acompanhamento de Ordens

---

## Descrição

> Como **servidora da secretaria**, quero que a tela "Minhas Ordens" seja exclusivamente para visualização e controle, para que fique claro que ações de workflow e upload de documentos acontecem somente nos pipelines específicos de cada etapa.

## Contexto e Motivação

A tela "Minhas Ordens" é o painel de acompanhamento geral da secretaria. Ações operacionais (atestar, reenviar, fazer upload de documentos) devem ocorrer nos pipelines dedicados de cada etapa, não nesta tela. Manter botões de ação aqui gera confusão sobre onde o processo deve ser executado.

---

## Critérios de Aceitação

### Cenário 1: Tela exibe ordens sem botões de ação
```gherkin
Given que estou autenticada com perfil "secretaria"
When acesso "Minhas Ordens"
Then vejo a lista de ordens da minha secretaria
And não há botões de upload, edição ou envio de documentos
And cada ordem exibe: protocolo, tipo, status, valor, data de criação
```

### Cenário 2: Documentos visíveis apenas para download
```gherkin
Given que acesso o detalhe de uma ordem em "Minhas Ordens"
When visualizo os documentos anexados
Then posso apenas visualizar e baixar documentos
And não há botão de upload ou exclusão de documentos
```

### Cenário 3: Histórico de tramitação disponível (leitura)
```gherkin
Given que acesso o detalhe de uma ordem
When visualizo o histórico de tramitação
Then vejo todos os eventos cronológicos
And não há nenhuma ação disponível no histórico
```

### Cenário 4: Ações de workflow existem somente nos pipelines
```gherkin
Given que tenho uma ordem com status "AGUARDANDO_ATESTO"
When acesso "Minhas Ordens"
Then não vejo botão "Atestar NF" nesta tela
And ao acessar o "Pipeline Atesto" vejo a ordem e posso executar a ação
```
