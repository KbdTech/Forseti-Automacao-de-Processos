# US-013 — Gestão de Secretarias pelo Administrador

**ID:** US-013
**Sprint:** S6 (Semanas 11–12)
**Status:** Ready
**Prioridade:** Média
**Depende de:** US-001, US-002
**Estimativa:** 5 pontos
**Módulo:** Administração

---

## Descrição

> Como **administrador do sistema**, quero gerenciar o cadastro de secretarias municipais (criar, editar e desativar), para que o sistema reflita a estrutura organizacional atual da prefeitura e as ordens sejam sempre vinculadas a secretarias válidas.

## Contexto e Motivação

O cadastro de secretarias é a base organizacional do sistema. Cada usuário do tipo "secretaria" é vinculado a uma secretaria, e cada ordem pertence a uma secretaria. O administrador precisa manter esse cadastro atualizado sem comprometer o histórico existente.

---

## Critérios de Aceitação

### Cenário 1: Listar todas as secretarias
```gherkin
Given que estou autenticado com perfil "admin"
When acesso a tela "Gestão de Secretarias"
Then vejo a lista de todas as secretarias (ativas e inativas)
And cada linha exibe: nome, sigla, orçamento anual, status (ativa/inativa)
And posso filtrar por status (ativas / inativas / todas)
```

### Cenário 2: Criar nova secretaria
```gherkin
Given que estou na tela de Gestão de Secretarias
When clico em "Nova Secretaria"
And preencho nome único, sigla única e orçamento anual
And confirmo
Then a secretaria é criada com status ativo
And aparece na lista imediatamente
```

### Cenário 3: Nome ou sigla duplicados são rejeitados
```gherkin
Given que já existe uma secretaria com nome "Secretaria de Saúde" ou sigla "SMS"
When tento criar uma nova secretaria com o mesmo nome ou mesma sigla
Then recebo o erro: "Nome de secretaria já cadastrado" ou "Sigla já em uso"
And nenhuma secretaria é criada
```

### Cenário 4: Editar dados de uma secretaria
```gherkin
Given que seleciono uma secretaria para editar
When altero o nome, sigla ou orçamento anual
And confirmo
Then os dados são atualizados no banco
And ordens existentes vinculadas a essa secretaria continuam funcionando normalmente
```

### Cenário 5: Desativar secretaria
```gherkin
Given que seleciono uma secretaria ativa
When clico em "Desativar" e confirmo o modal de confirmação
Then a secretaria é marcada como inativa (ativo = false)
And a secretaria não aparece mais no select de "Nova Ordem" para criação
And o histórico e todas as ordens existentes da secretaria são preservados
```

### Cenário 6: Não é possível excluir secretaria
```gherkin
Given que seleciono qualquer secretaria
When verifico as opções disponíveis
Then não existe botão de "Excluir" — apenas "Desativar"
And o endpoint DELETE /api/secretarias/:id não existe (retorna 404 ou 405)
```

### Cenário 7: Orçamento anual pode ser editado a qualquer momento
```gherkin
Given que uma secretaria está com orçamento de R$ 500.000,00
When o Admin edita o orçamento para R$ 750.000,00
Then o novo valor é salvo imediatamente
And o histórico de ordens não é afetado
```

## Regras de Negócio

| ID | Regra |
|----|-------|
| RN-65 | Nome e sigla da secretaria devem ser únicos no sistema |
| RN-66 | Secretaria desativada não pode receber novas ordens, mas mantém histórico |
| RN-67 | Orçamento anual pode ser editado a qualquer momento pelo Administrador |
| RN-68 | Não é possível excluir uma secretaria — apenas desativar |

## Escopo

### IN ✅
- Tela de listagem de secretarias com filtro de status
- Modal de criação com validação de nome/sigla únicos
- Modal de edição de dados (nome, sigla, orçamento)
- Toggle de ativação/desativação com modal de confirmação
- Secretarias inativas excluídas do select de criação de ordens
- CRUD completo via /api/secretarias (sem DELETE)

### OUT ❌
- Exclusão física de secretarias
- Gestão de usuários por secretaria (coberta em US-002/Admin)
- Transferência de ordens entre secretarias
- Histórico de mudanças de orçamento
