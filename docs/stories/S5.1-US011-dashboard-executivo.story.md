## Descrição

> Como **membro do Gabinete ou administrador**, quero visualizar um dashboard executivo com KPIs, gráficos do funil de aprovação e alertas automáticos de gargalos, para que eu possa monitorar a saúde do processo de compras públicas e tomar decisões baseadas em dados.

## Contexto e Motivação

O dashboard é a visão estratégica do sistema. Consolida dados de todas as etapas do fluxo em KPIs e visualizações que permitem identificar rapidamente onde o processo está travado, quais secretarias têm mais devoluções e quanto do orçamento já foi comprometido.

---

## Critérios de Aceitação

### Cenário 1: KPIs principais são exibidos corretamente
```gherkin
Given que estou autenticado com perfil "gabinete" ou "admin"
When acesso o Dashboard Executivo
Then vejo os KPIs: Total de Ordens, Valor Total (R$), Em Aberto, Pagas, Taxa de Reprovação (%), Tempo Médio de Processo (dias)
And os valores refletem o período selecionado (padrão: mês atual)
And os dados são calculados no servidor — não no front-end
```

### Cenário 2: Gráfico de funil exibe distribuição por status
```gherkin
Given que acesso o Dashboard
When visualizo o FunnelChart
Then vejo a quantidade de ordens em cada etapa do fluxo (AGUARDANDO_GABINETE → PAGA)
And o funil evidencia onde há mais ordens represadas
```

### Cenário 3: Gráfico de barras por secretaria
```gherkin
Given que acesso o Dashboard
When visualizo o BarChart
Then vejo o volume de ordens por secretaria no período
And posso identificar quais secretarias geram mais demandas
```

### Cenário 4: Alertas automáticos de gargalos
```gherkin
Given que existem ordens paradas na mesma etapa há mais de 5 dias úteis
When acesso a seção de alertas do Dashboard
Then cada ordem em gargalo aparece listada com: protocolo, secretaria, etapa, há quantos dias úteis está parada
And cada alerta tem um link clicável que leva ao detalhe da ordem
```

### Cenário 5: Alerta de secretaria com alta taxa de devolução
```gherkin
Given que uma secretaria tem taxa de devolução/irregularidade acima de 20% no período
When acesso os alertas do Dashboard
Then essa secretaria aparece com alerta de atenção em destaque
And o percentual de devolução é exibido ao lado do nome da secretaria
```

### Cenário 6: Filtro de período com DateRangePicker
```gherkin
Given que estou no Dashboard
When seleciono um período personalizado com DateRangePicker
Then todos os KPIs e gráficos são recalculados para o período informado
And o período máximo aceito é 12 meses
And o botão de refresh manual recarrega os dados mantendo o período atual
```

### Cenário 7: Dados atualizados automaticamente
```gherkin
Given que estou com o Dashboard aberto
When passam 5 minutos sem interação
Then os dados são recarregados automaticamente
And um indicador exibe "Atualizado às HH:MM" após cada refresh
```

## Escopo

### IN ✅
- Endpoint GET /api/dashboard/summary com KPIs e dados de gráficos (queries GROUP BY)
- Endpoint GET /api/dashboard/alertas com gargalos (> 5 dias úteis) e secretarias com > 20% devolução
- FunnelChart (distribuição por status), BarChart (volume por secretaria), StackedBarChart (opcional)
- DateRangePicker com período máximo de 12 meses
- Auto-refresh a cada 5 minutos
- Botão de refresh manual
- Links clicáveis nos alertas para o detalhe da ordem
- Somente perfis gabinete e admin acessam o dashboard executivo completo

### OUT ❌
- Export de dados do dashboard em PDF/Excel
- Dashboard personalizado por usuário
- Dados históricos além de 12 meses no filtro
- Notificações push de gargalos
