# US-008 — Registro de Empenho pela Contabilidade

**ID:** US-008
**Sprint:** S4 (Semanas 7–8)
**Status:** Ready
**Prioridade:** Alta
**Depende de:** US-007
**Estimativa:** 8 pontos
**Módulo:** Pipeline Operacional (Financeiro)

---

## Descrição

> Como **contador da Contabilidade**, quero registrar o empenho orçamentário de uma ordem aprovada pela Controladoria, para que o valor seja formalmente reservado no orçamento municipal e a ordem avance para a fase de execução.

## Contexto e Motivação

O empenho é o ato que compromete o orçamento municipal. Após a aprovação da Controladoria, a Contabilidade registra o número do empenho (único no sistema) e o valor empenhado, que pode diferir do valor estimado. A partir deste momento, a ordem não pode mais ser cancelada sem processo especial.

```gherkin

When acesso a tela "Pipeline Empenho"
Then vejo todas as ordens com status "AGUARDANDO_EMPENHO"
```

### Cenário 2: Registrar empenho com dados válidos
```gherkin
Given que seleciono uma ordem com status "AGUARDANDO_EMPENHO"
When preencho o número do empenho e o valor empenhado
And clico em "Registrar Empenho" e confirmo
Then a ordem avança para status "AGUARDANDO_EXECUCAO"
And os campos numero_empenho, valor_empenhado e data_empenho são gravados na ordem
And a data do empenho é registrada automaticamente (data atual)
And um registro em ordem_historico: acao="empenhar", perfil="contabilidade"
```

### Cenário 3: Número de empenho duplicado é rejeitado
```gherkin
Given que existe uma ordem com numero_empenho "EMP-2026-001"
When tento registrar outro empenho com o mesmo número "EMP-2026-001"
Then recebo HTTP 422: "Número de empenho já cadastrado no sistema"
And nenhuma alteração é feita no banco
```

### Cenário 4: Valor empenhado diferente do estimado
```gherkin
Given que o valor estimado da ordem é R$ 10.000,00
When registro um empenho com valor_empenhado de R$ 9.500,00
Then a ordem é empenhada com sucesso
And ambos os valores (estimado e empenhado) ficam registrados na ordem
And o sistema não bloqueia diferença entre os valores (apenas registra)
```

### Cenário 5: Campos obrigatórios — número de empenho e valor
```gherkin
Given que estou no modal de empenho
When submeto sem preencher o número do empenho ou o valor
Then o formulário exibe: "Número do empenho é obrigatório" / "Valor empenhado é obrigatório"
And nenhuma requisição é enviada ao back-end
```

### Cenário 6: Após empenho, ordem não pode ser cancelada
```gherkin
Given que uma ordem está com status "AGUARDANDO_EXECUCAO"
When qualquer usuário tenta cancelar a ordem
Then recebo HTTP 422: "Ordens empenhadas não podem ser canceladas sem processo de desempenho"
And a ordem permanece no status atual
```

## Regras de Negócio

| ID | Regra |
|----|-------|
| RN-42 | Número do empenho é obrigatório e deve ser único no sistema |
| RN-43 | Data do empenho é registrada automaticamente (data do sistema) |
| RN-44 | Após o empenho, a ordem não pode ser cancelada sem processo especial de desempenho |
| RN-45 | O valor empenhado pode ser diferente do valor estimado — deve ser registrado |
