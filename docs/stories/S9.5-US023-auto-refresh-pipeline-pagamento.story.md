# US-023 — Auto-refresh e Botão de Refresh no Pipeline de Pagamento

**ID:** US-023
**Sprint:** S9
**Status:** Done
**Prioridade:** Média
**Depende de:** US-010, US-019
**Módulo:** Frontend / PagamentoPage + WorkflowTable

---

## Descrição

> Como **usuário com perfil tesouraria**, quero que o Pipeline de Pagamento atualize automaticamente a cada 30 segundos e tenha um botão de refresh manual com indicador de "última atualização", para que eu não precise recarregar a página para ver novas ordens chegando na fila após a secretaria assinar a liquidação.

## Contexto e Motivação

O servidor da tesouraria mantém `PagamentoPage` aberta durante o expediente. Quando uma secretaria assina a liquidação (US-019) e a ordem avança para `AGUARDANDO_PAGAMENTO`, ela não aparece até o servidor recarregar a página manualmente. O ícone de refresh circular (mesmo estilo já existente em outros pontos do sistema) deve aparecer no cabeçalho junto com o indicador de tempo desde a última atualização.

---

## Estado Atual do Código

**`PagamentoPage.tsx`** (`frontend/src/pages/tesouraria/PagamentoPage.tsx`) é minimalista — não possui estado de dados próprio. Todo o fetch é gerenciado por **`WorkflowTable`** (`frontend/src/components/workflow/WorkflowTable.tsx`), que internamente usa:

```tsx
const { data, isLoading, isError, refetch } = useQuery({
  queryKey: ['ordens', 'workflow', statusParam, page, ...],
  staleTime: 1000 * 30,
})
```

`WorkflowTable` já expõe `refetch` internamente mas **não o passa para fora via props**. A solução correta é adicionar uma prop `autoRefreshMs` ao `WorkflowTable` que ativa polling interno, e uma prop `onRefreshReady` que passa o `refetch` para a página pai poder acionar o botão manual.

```gherkin
Given que estou na tela de Pipeline de Pagamento com a lista carregada

```

### Cenário 2: Botão de refresh manual
```gherkin
Given que estou no Pipeline de Pagamento
When visualizo o cabeçalho da página
Then existe um botão com ícone de refresh circular (RefreshCw do lucide-react)
And ao clicar, a lista é recarregada imediatamente
And o indicador "Atualizado há X segundos" é zerado para "Atualizado agora"
```

### Cenário 3: Indicador de última atualização
```gherkin
Given que a lista foi carregada (por auto-refresh ou clique manual)
Then o cabeçalho exibe "Atualizado agora" logo após a carga
And o texto muda para "Atualizado há Xs" incrementando a cada segundo
```

### Cenário 4: Polling não interfere com ações em andamento
```gherkin
Given que estou com o modal de detalhes/pagamento aberto
When o timer de 30 segundos dispara
Then o auto-refresh NÃO é executado (polling pausado com modal aberto)
And ao fechar o modal, o polling é retomado normalmente
```

### Cenário 5: Nova ordem aparece sem reload manual
```gherkin
Given que estou no Pipeline de Pagamento sem ordens visíveis
And uma secretaria assina a liquidação de uma ordem em outra sessão
When passam até 30 segundos

```

## Especificação Técnica

### Estratégia de implementação

A solução passa pelo `WorkflowTable` (componente compartilhado), adicionando props opcionais para não quebrar nenhuma outra página que o usa:

---

### 1. Novas props em `WorkflowTable`

**Arquivo:** `frontend/src/components/workflow/WorkflowTable.tsx`

```tsx
interface WorkflowTableProps {
  // ... props existentes mantidas ...

  /** Intervalo em ms para auto-refresh. Undefined = sem polling. */
  autoRefreshMs?: number
  /** Callback chamado após cada refetch (manual ou automático). Recebe o timestamp. */
  onRefresh?: (timestamp: Date) => void
  /** Quando true, pausa o polling (ex.: modal aberto na página pai). */
  pausePolling?: boolean
}
```

**Polling interno no `WorkflowTable`** (adicionar após o `useQuery` existente):
```tsx
// Auto-refresh com polling — pausável
useEffect(() => {
  if (!autoRefreshMs || pausePolling) return
  const id = setInterval(async () => {
    await refetch()
    onRefresh?.(new Date())
  }, autoRefreshMs)
  return () => clearInterval(id)
}, [autoRefreshMs, pausePolling, refetch, onRefresh])
```

---

### 2. Atualizar `PagamentoPage`

**Arquivo:** `frontend/src/pages/tesouraria/PagamentoPage.tsx`

```tsx
import { useState, useCallback } from 'react'
import { RefreshCw } from 'lucide-react'
import { Button } from '@/components/ui/button'
import { POLLING_INTERVAL_MS } from '@/utils/constants'

export default function PagamentoPage() {
  const [lastUpdated, setLastUpdated] = useState<Date>(new Date())
  const [isModalOpen, setIsModalOpen] = useState(false)
  const [manualRefreshFn, setManualRefreshFn] = useState<(() => void) | null>(null)

  const handleRefresh = useCallback((timestamp: Date) => {
    setLastUpdated(timestamp)
  }, [])

  return (
    <div className="container max-w-7xl mx-auto py-8 px-4 space-y-6">
      <div className="flex items-center justify-between">
        <div>
          <h1 className="text-2xl font-bold">Registro de Pagamento</h1>
          <p className="text-muted-foreground text-sm mt-1">
            Ordens liquidadas aguardando confirmação de pagamento pela Tesouraria.
          </p>
        </div>
        <div className="flex items-center gap-3">
          <span className="text-xs text-muted-foreground">
            {formatRelativeTime(lastUpdated)}
          </span>
          <Button
            variant="outline"
            size="sm"
            className="gap-1.5"
            onClick={() => manualRefreshFn?.()}
          >
            <RefreshCw className="h-4 w-4" />
            Atualizar
          </Button>
        </div>
      </div>

      <WorkflowTable
        statusFilter="AGUARDANDO_PAGAMENTO"
        title="Ordens aguardando pagamento"
        emptyMessage="Nenhuma ordem aguardando pagamento no momento."
        autoRefreshMs={POLLING_INTERVAL_MS}
        pausePolling={isModalOpen}
        onRefresh={handleRefresh}
        renderActions={(orderId, _status, onActionComplete) => (
          <PagamentoActionPanel
            orderId={orderId}
            onSuccess={onActionComplete}
            onModalOpen={() => setIsModalOpen(true)}
            onModalClose={() => setIsModalOpen(false)}
          />
        )}
      />
    </div>
  )
}
```

### 3. Novas constante e helper

**Arquivo:** `frontend/src/utils/constants.ts`
```tsx
/** Intervalo de polling para pipelines em tempo real (ms). */
export const POLLING_INTERVAL_MS = 30_000
```

**Arquivo:** `frontend/src/utils/formatters.ts`
```tsx
/** Retorna texto relativo desde a última atualização. */
export function formatRelativeTime(date: Date): string {
  const seconds = Math.floor((Date.now() - date.getTime()) / 1000)
  if (seconds < 5) return 'Atualizado agora'
  return `Atualizado há ${seconds}s`
}
```

> Para o indicador incrementar a cada segundo, usar um `setInterval` de 1s dentro de `PagamentoPage` para forçar re-render do texto (ou um hook `useNow` simples).
