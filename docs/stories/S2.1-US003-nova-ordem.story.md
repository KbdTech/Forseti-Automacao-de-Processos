# US-003 — Criação de Nova Ordem de Serviço ou Compra

**ID:** US-003
**Sprint:** S2 (Semanas 3–4)
**Status:** Ready
**Prioridade:** Alta
**Depende de:** US-001, US-002
**Estimativa:** 13 pontos
**Módulo:** Criação e Acompanhamento de Ordens

---

## Descrição

> Como **servidor de uma secretaria municipal**, quero criar uma nova Ordem de Serviço ou Compra em um formulário estruturado, para que a demanda seja formalizada, protocolada e encaminhada automaticamente para aprovação do Gabinete.

## Contexto e Motivação

Esta é a porta de entrada do fluxo principal. Toda ordem nasce aqui: com tipo, prioridade, valor estimado e justificativa. O protocolo é gerado automaticamente no padrão OS-ANO-SEQUENCIAL. Substitui o formulário mockado por persistência real no PostgreSQL.

---

## Critérios de Aceitação

### Cenário 1: Criação de ordem com dados válidos
```gherkin
Given que estou autenticado com perfil "secretaria"
And estou na tela "Nova Ordem"
When preencho todos os campos obrigatórios com dados válidos
And clico em "Confirmar e Enviar"
Then a ordem é criada no banco com status "AGUARDANDO_GABINETE"
And recebo um número de protocolo no formato OS-2026-XXXXX
And a secretaria de origem é vinculada automaticamente ao meu usuário
And sou redirecionado para a tela de confirmação com o protocolo exibido
And um registro é inserido em ordem_historico com ação "criar"
```

### Cenário 2: Validação de campos obrigatórios
```gherkin
Given que estou na tela "Nova Ordem"
When tento avançar na etapa sem preencher campos obrigatórios
Then o formulário exibe mensagens de validação por campo
And não é possível avançar para a próxima etapa
And nenhuma requisição é enviada ao back-end
```

### Cenário 3: Justificativa com menos de 50 caracteres
```gherkin
Given que estou na etapa 2 do formulário
When digito uma justificativa com menos de 50 caracteres
Then o campo exibe: "Justificativa deve ter no mínimo 50 caracteres (X/50)"
And o botão "Próxima etapa" permanece desabilitado
```

### Cenário 4: Valor estimado inválido
```gherkin
Given que estou na etapa 2 do formulário
When informo um valor estimado menor ou igual a zero
Then o campo exibe: "O valor deve ser maior que zero"
And não é possível avançar
```

### Cenário 5: Protocolo gerado é único
```gherkin
Given que dois usuários criam ordens simultaneamente
When ambas as criações são processadas
Then cada ordem recebe um protocolo único e sequencial
And nenhum protocolo é duplicado
```

### Cenário 6: Perfil não-secretaria não pode criar ordem
```gherkin
Given que estou autenticado com perfil "gabinete"
When tento acessar a rota /nova-ordem ou chamar POST /api/ordens
Then recebo HTTP 403 "Acesso negado para este perfil"
And nenhuma ordem é criada
```

## Regras de Negócio

| ID | Regra |
|----|-------|
| RN-13 | Protocolo gerado automaticamente no padrão OS-ANO-SEQUENCIAL (ex: OS-2026-00001) |
| RN-14 | Data e hora de criação registradas automaticamente |
| RN-15 | Secretaria de origem vinculada automaticamente ao usuário criador |
| RN-16 | Tipo de ordem: COMPRA, SERVIÇO ou OBRA (obrigatório) |
| RN-17 | Prioridade: NORMAL, ALTA ou URGENTE (obrigatório) |
| RN-18 | Valor estimado é obrigatório e deve ser um número positivo |
| RN-19 | Justificativa mínima de 50 caracteres |
| RN-20 | Ao criar, o status deve ser AGUARDANDO_GABINETE automaticamente |

## Escopo

### IN ✅
- Formulário StepperForm de 3 etapas com validação por etapa
- Etapa 1: Tipo de Ordem + Prioridade + Responsável (opcional)
- Etapa 2: Valor Estimado + Descrição (opcional) + Justificativa (min 50 chars)
- Etapa 3: Revisão completa + confirmação
- Geração atômica de protocolo OS-ANO-SEQUENCIAL
- Vínculo automático de secretaria_id do usuário logado
- Registro em ordem_historico na criação
- Tela de sucesso exibindo o protocolo gerado

### OUT ❌
- Upload de anexos (não coberto nesta US)
- Duplicação de ordens existentes
- Rascunho salvo localmente (sem persistência parcial)
- Criação de ordens por outros perfis além de secretaria
