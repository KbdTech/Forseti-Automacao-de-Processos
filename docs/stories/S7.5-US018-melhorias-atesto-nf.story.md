# US-018 — Melhorias no Atesto de Nota Fiscal (Secretaria)

**ID:** US-018
**Sprint:** S7
**Status:** Ready
**Prioridade:** Alta
**Depende de:** US-009, US-015, US-016
**Módulo:** Pipeline Operacional (Financeiro)

---

## Descrição

> Como **servidora da secretaria**, quero anexar a nota fiscal e documentos extras ao atestar o recebimento do serviço/compra, e confirmar que a DLD foi assinada, para que o atesto tenha comprovação documental completa e conformidade com as exigências legais.

## Contexto e Motivação

O atesto da nota fiscal é o momento em que a secretaria confirma que o serviço foi executado ou o produto entregue. Para garantir a conformidade, além do número da NF (já existente), é necessário anexar o arquivo da NF, permitir documentos extras, e confirmar a assinatura da DLD (Declaração de Liquidação e Despesa) antes de confirmar o atesto.

## Especificação Técnica

### Front-End

**Modal de Atesto — campos e validação:**
```typescript
interface AtesteFormData {
  numero_nf: string           // obrigatório
  arquivo_nf: File            // obrigatório
  documentos_extras: File[]   // opcional, múltiplos
  observacao: string          // opcional
  servico_executado: boolean  // obrigatório — deve ser true
  dld_assinada: boolean       // obrigatório — deve ser true
}

// Botão habilitado somente quando:
const podeConfirmar =
  numero_nf.trim() !== '' &&
  arquivo_nf !== null &&
  servico_executado === true &&
  dld_assinada === true
```

**Componente de alerta para DLD:**
- Usar `Alert` do shadcn/ui com ícone de atenção
- Texto: *"Confirme que a Declaração de Liquidação e Despesa (DLD) foi assinada antes de prosseguir."*
- Checkbox dentro do Alert com label: *"A DLD foi assinada?"*

**Fluxo de envio:**
1. Upload da NF: `documentosService.upload(ordemId, arquivo_nf, 'NOTA_FISCAL')`
2. Upload extras (se houver): `documentosService.upload(ordemId, file, 'DOCUMENTO_ATESTO')` para cada
3. `PATCH /api/ordens/{id}/acao` com `{ acao: 'atestar', numero_nf, observacao }`
4. `onClose()` + `onSuccess()` (UX-001)

### Back-End

**Ação `atestar` em `PATCH /api/ordens/{id}/acao`:**
- Adicionar campo `observacao` (opcional) ao payload de atesto
- Registrar `observacao` no `ordem_historico`
- Nenhuma validação adicional no backend para a DLD (confirmação é responsabilidade do front)

**Tipos de documento:** `NOTA_FISCAL`, `DOCUMENTO_ATESTO` — garantir que estão mapeados.
