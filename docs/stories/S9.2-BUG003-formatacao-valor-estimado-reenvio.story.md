# BUG-003 — Valor Estimado com formatação incorreta ao editar ordem devolvida

**ID:** BUG-003
**Sprint:** S9
**Status:** Done
**Prioridade:** Alta
**Depende de:** US-006, BUG-001 (S7)
**Módulo:** Frontend / EditarOrdemPage

---

## Descrição

> Como **usuário com perfil secretaria**, ao abrir a página de edição de uma ordem devolvida, quero que o campo "Valor Estimado" já apareça formatado corretamente (ex.: `R$ 10.000,00`) para que eu possa confirmar ou editar o valor sem precisar corrigir a formatação manualmente antes de reenviar.

## Contexto e Motivação

Quando uma ordem é devolvida e o usuário navega para `/secretaria/ordens/:id/editar`, o campo `valor_estimado` exibe `10000.00` (formato americano, ponto no lugar de vírgula) com borda vermelha e erro **"Informe um valor válido."** — bloqueando o reenvio.

O bug é **duplo**:
1. **Display:** `valorInput` é inicializado com `ordem.valor_estimado.toLocaleString('pt-BR', { style: 'currency' })`, mas se a API retornar o valor como **string** (`"10000.00"`), `String.prototype.toLocaleString` ignora as opções de currency e devolve a string crua.
2. **Zod:** o `reset()` do React Hook Form inicializa `valor_estimado` com o valor bruto da API. Se vier como string, o schema `z.number({ invalid_type_error: 'Informe um valor válido.' })` rejeita imediatamente e o campo fica com estado de erro ao abrir — sem o usuário ter tocado em nada.

## Especificação Técnica

### Front-End — único arquivo a modificar

**Arquivo:** `frontend/src/pages/secretaria/EditarOrdemPage.tsx`

**Correção no `useEffect` de pré-preenchimento (~linha 241) — duas mudanças:**

```tsx
useEffect(() => {
  if (!ordem) return

  if (ordem.status !== 'DEVOLVIDA_PARA_ALTERACAO') {
    toast.error('Ordem não disponível para edição', {
      description: 'Apenas ordens devolvidas pelo Gabinete podem ser editadas.',
    })
    navigate('/secretaria/devolvidas')
    return
  }

  // CORREÇÃO 1: Garantir cast para Number antes de formatar o display
  const numericValue = Number(ordem.valor_estimado)
  const formatted = isNaN(numericValue)
    ? ''
    : numericValue.toLocaleString('pt-BR', { style: 'currency', currency: 'BRL' })
  setValorInput(formatted)

  reset({
    tipo: ordem.tipo,
    prioridade: ordem.prioridade,
    responsavel: ordem.responsavel ?? '',
    assinatura_govbr: ordem.assinatura_govbr ?? false,
    descricao: ordem.descricao ?? '',
    // CORREÇÃO 2: Garantir que o form recebe number, não string
    valor_estimado: numericValue,
    justificativa: ordem.justificativa,
  })
}, [ordem, reset, navigate])
```

> **Importante:** `EditarOrdemPage.tsx` possui helpers `formatBRL` e `parseBRL` **próprios** (definidos localmente, ~linhas 125–135). Não importar de `formatters.ts` — manter os helpers locais do arquivo.
