## Descrição

> Como **tesoureira**, ao registrar o pagamento de uma ordem vinculada a um fornecedor, quero visualizar os dados bancários da empresa diretamente no modal de pagamento, para evitar erros de digitação e reduzir o tempo necessário para localizar as informações bancárias.

## Contexto e Motivação

Atualmente, a Tesouraria precisa buscar manualmente os dados bancários do fornecedor em outra fonte para preencher o comprovante de transferência/PIX. Com os dados bancários cadastrados no sistema (S11.1) e o fornecedor sendo **obrigatório** em todas as novas ordens (S11.3), o card de dados bancários estará presente em praticamente todas as ordens em pagamento. O card exibe os dados como referência — sem preencher campos automaticamente, pois o valor e forma de pagamento são definidos pelo usuário no momento do pagamento.

## Critérios de Aceitação

### Cenário 1: Card de dados bancários exibido quando fornecedor vinculado

```gherkin
Given que a ordem em "AGUARDANDO_PAGAMENTO" tem fornecedor vinculado com dados bancários cadastrados
When a tesoureira abre o modal de pagamento
Then vejo um card "Dados Bancários do Fornecedor" ACIMA dos campos de pagamento com:
  - Badge "Fornecedor Vinculado" com razão social e CNPJ formatado
  - Banco: {nome do banco}
  - Agência: {agencia}
  - Conta: {conta} ({tipo_conta — Corrente ou Poupança})
And os dados são exibidos em modo somente-leitura (não são inputs)
```

### Cenário 2: Card exibido antes dos campos de pagamento

```gherkin
Given que a ordem tem fornecedor com dados bancários
Then a ordem visual dentro do modal é:
  1. Card "Dados Bancários do Fornecedor" (topo)
  2. Campo Valor Pago
  3. Campo Data do Pagamento
  4. Campo Forma de Pagamento
  5. Upload Comprovante
  6. Botão "Registrar Pagamento"
```

### Cenário 3: Card NÃO exibido quando sem fornecedor ou sem dados bancários

```gherkin
Given que a ordem NÃO tem fornecedor vinculado
Then o card "Dados Bancários do Fornecedor" NÃO aparece
And o modal funciona exatamente como antes (sem alteração)

Given que a ordem tem fornecedor mas sem dados bancários (banco, agencia, conta = null)
Then o card é exibido apenas com a identificação do fornecedor (razão social + CNPJ)
And campos bancários ausentes são omitidos do card (não exibir "Banco: —")
```

### Cenário 4: Dados bancários são informativos — não preenchem formulário

```gherkin
Given que o card exibe os dados bancários
Then os campos do formulário (valor, data, forma_pagamento) permanecem em branco
And a tesoureira preenche manualmente os campos do formulário
And os dados do card são apenas referência visual
```

### Cenário 5: Fluxo completo de pagamento com fornecedor vinculado

```gherkin
Given que a ordem tem fornecedor com dados bancários
And a tesoureira visualizou os dados bancários do card
And preencheu valor_pago, data_pagamento, forma_pagamento e anexou comprovante
When clica em "Registrar Pagamento"
Then a ordem avança para status PAGA
And todos os dados do pagamento são registrados normalmente
And o card de dados bancários não interfere no fluxo
```

## Especificação Técnica

### Frontend

**Arquivo:** `frontend/src/components/orders/PagamentoModal.tsx`

**Sem novos imports necessários** — `formatCNPJ` de `utils/formatters.ts` (adicionado em S11.2).

**Card de dados bancários** (inserir antes do campo `valor_pago`):

```tsx
{/* Card de dados bancários do fornecedor — S12.1 */}
{ordem?.fornecedor && (
  <div className="rounded-lg border bg-muted/30 p-4 space-y-3">
    {/* Badge de identificação */}
    <div className="flex items-start gap-2">
      <Badge variant="outline" className="text-xs shrink-0">
        Fornecedor Vinculado
      </Badge>
      <div>
        <p className="text-sm font-medium leading-tight">{ordem.fornecedor.razao_social}</p>
        <p className="text-xs text-muted-foreground">{formatCNPJ(ordem.fornecedor.cnpj)}</p>
      </div>
    </div>

    {/* Dados bancários — exibe apenas se preenchidos */}
    {(ordem.fornecedor.banco || ordem.fornecedor.agencia || ordem.fornecedor.conta) && (
      <div className="grid grid-cols-2 gap-x-4 gap-y-1 text-sm border-t pt-3">
        <p className="text-muted-foreground text-xs font-medium col-span-2">Dados Bancários</p>
        {ordem.fornecedor.banco && (
          <>
            <span className="text-muted-foreground text-xs">Banco</span>
            <span className="text-xs">{ordem.fornecedor.banco}</span>
          </>
        )}
        {ordem.fornecedor.agencia && (
          <>
            <span className="text-muted-foreground text-xs">Agência</span>
            <span className="text-xs">{ordem.fornecedor.agencia}</span>
          </>
        )}
        {ordem.fornecedor.conta && (
          <>
            <span className="text-muted-foreground text-xs">Conta</span>
            <span className="text-xs">
              {ordem.fornecedor.conta}
              {' '}
              <span className="text-muted-foreground">
                ({ordem.fornecedor.tipo_conta === 'poupanca' ? 'Poupança' : 'Corrente'})
              </span>
            </span>
          </>
        )}
      </div>
    )}
  </div>
)}
```
