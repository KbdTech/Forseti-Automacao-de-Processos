# UX-001 — Fechar Modal Automaticamente Após Ação Confirmada

**ID:** UX-001
**Sprint:** S7
**Status:** Ready
**Prioridade:** Crítica
**Depende de:** —
**Módulo:** Global (todos os modais de ação)

---

## Descrição

> Como **usuário do sistema**, quero que o modal feche automaticamente após confirmar uma ação com sucesso, para que eu não precise fechá-lo manualmente enquanto o processo já avançou para o próximo passo.

## Contexto e Motivação

Atualmente após ações de workflow (empenhar, atestar, liquidar, pagar, autorizar, devolver etc.) o modal permanece aberto mesmo após a API retornar sucesso. O usuário precisa clicar em fechar manualmente, o que gera confusão sobre se a ação foi processada ou não.

---

## Critérios de Aceitação

### Cenário 1: Modal fecha após ação bem-sucedida
```gherkin
Given que estou com um modal de ação aberto (ex: Empenho)
When preencho os dados e clico em confirmar
And a API retorna status 200
Then o modal fecha automaticamente
And a lista/pipeline é atualizada (item removido ou status atualizado)
And um toast de sucesso é exibido
```

### Cenário 2: Modal permanece aberto em caso de erro
```gherkin
Given que estou com um modal de ação aberto
When preencho os dados e clico em confirmar
And a API retorna status 4xx ou 5xx
Then o modal permanece aberto
And um toast de erro é exibido com a mensagem retornada pela API
And os dados preenchidos são mantidos (não resetar o formulário)
```

### Cenário 3: Comportamento uniforme em todos os modais
```gherkin
Given qualquer modal de ação de workflow
When a ação é confirmada com sucesso
Then o comportamento de fechar + atualizar lista é sempre o mesmo
```

## Especificação Técnica

### Front-End

**Padrão a aplicar em todos os modais:**
```typescript
const handleConfirmar = async () => {
  try {
    setLoading(true)
    await executeAcao(ordemId, payload)
    onClose()           // fecha modal
    onSuccess()         // refetch da lista / remove item
    toast.success('Ação realizada com sucesso')
  } catch (error) {
    toast.error(getErrorMessage(error))
    // modal permanece aberto
  } finally {
    setLoading(false)
  }
}
```

**Props necessárias nos modais:**
- `onClose: () => void` — já deve existir na maioria
- `onSuccess: () => void` — callback para atualizar a lista pai

### Back-End

Sem alterações necessárias.
