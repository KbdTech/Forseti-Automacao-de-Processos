# US-017 — Upload Obrigatório de Documento de Empenho (Contabilidade)

**ID:** US-017
**Sprint:** S7
**Status:** Ready
**Prioridade:** Alta
**Depende de:** US-008, US-015
**Módulo:** Pipeline Operacional (Financeiro)

---

## Descrição

> Como **contadora da Contabilidade**, quero anexar o documento de empenho ao registrar o empenho de uma ordem, para que o ato orçamentário tenha comprovação documental vinculada à ordem no sistema.

## Contexto e Motivação

O empenho é um ato formal que compromete o orçamento municipal. O documento de empenho (nota de empenho emitida pelo sistema contábil da prefeitura) deve ser anexado obrigatoriamente no momento do registro, garantindo rastreabilidade documental completa.

---

## Critérios de Aceitação

### Cenário 1: Modal de empenho exige upload de documento
```gherkin
Given que estou autenticada com perfil "contabilidade"
And acesso o Pipeline Empenho com uma ordem em "AGUARDANDO_EMPENHO"
When abro o modal de empenho
Then vejo os campos: número do empenho, valor empenhado (pré-preenchido + formatado) e upload de documento
And o botão "Registrar Empenho" está desabilitado enquanto nenhum arquivo estiver anexado
```

### Cenário 2: Registrar empenho com documento anexado
```gherkin
Given que preenchi número do empenho, valor empenhado e anexei o documento
When clico em "Registrar Empenho" e confirmo
Then a ordem avança para status "AGUARDANDO_ATESTO"
And o documento é salvo com tipo "EMPENHO" vinculado à ordem
And um registro em ordem_historico: acao="empenhar", perfil="contabilidade"
And o modal fecha automaticamente (UX-001)
And o item sai do pipeline de empenho
```

### Cenário 3: Não é possível empenhar sem documento
```gherkin
Given que preenchi número e valor mas não anexei documento
When tento clicar em "Registrar Empenho"
Then o botão permanece desabilitado
And exibo mensagem: "Anexe o documento de empenho para continuar"
```

### Cenário 4: Valor empenhado pré-preenchido e formatado
```gherkin
Given que a ordem tem valor_estimado de R$ 15.000,00
When abro o modal de empenho
Then o campo "Valor Empenhado" exibe "R$ 15.000,00"
And posso editar sem precisar apagar vírgula ou decimais (BUG-001)
```

## Especificação Técnica

### Back-End

**Alteração em `PATCH /api/ordens/{id}/acao`** — ação `empenhar`:
- Aceitar `multipart/form-data` OU continuar JSON com upload feito separadamente via `POST /api/ordens/{id}/documentos`
- **Fluxo recomendado:** upload do documento primeiro (`POST /api/ordens/{id}/documentos`), depois `PATCH acao=empenhar`
- Validar que existe pelo menos 1 documento do tipo `EMPENHO` antes de aceitar a transição (opcional: backend valida, ou frontend garante)

**Tipo de documento:** `EMPENHO` — adicionar ao enum/constante de tipos se necessário.

### Front-End

**Modal de Empenho:**
```typescript
// Campos:
// - numero_empenho: string (obrigatório)
// - valor_empenhado: string formatada → parseCurrency antes de enviar
// - documento: File (obrigatório)

// Fluxo:
// 1. Upload do documento via documentosService.upload(ordemId, file, 'EMPENHO')
// 2. PATCH acao='empenhar' com numero_empenho e valor_empenhado
// 3. onClose() + onSuccess() em caso de sucesso (UX-001)
```

**Botão desabilitado enquanto:**
- `numero_empenho` vazio, OU
- `valor_empenhado` vazio/inválido, OU
- nenhum arquivo selecionado
