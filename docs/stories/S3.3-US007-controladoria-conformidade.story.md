# US-007 — Análise de Conformidade pela Controladoria

**ID:** US-007
**Sprint:** S3 (Semanas 5–6)

**Estimativa:** 8 pontos

---

## Descrição

> Como **fiscal da Controladoria**, quero analisar a conformidade legal e fiscal das ordens aprovadas pelo Gabinete e emitir parecer (aprovação, irregularidade ou solicitação de documentação), para que apenas ordens legalmente conformes avancem para o processo de empenho.

## Contexto e Motivação

A Controladoria é o guardião legal do processo. Após o Gabinete autorizar, a Controladoria verifica se a demanda está em conformidade com a legislação de compras públicas. Tem poder de aprovar, apontar irregularidade (suspende a ordem) ou solicitar documentação adicional.

## Critérios de Aceitação

### Cenário 1: Visualizar fila de ordens aguardando análise
```gherkin
Given que estou autenticado com perfil "controladoria"
When acesso a tela "Pipeline Controladoria"
Then vejo todas as ordens com status "AGUARDANDO_CONTROLADORIA" ou "AGUARDANDO_DOCUMENTACAO"
And cada linha exibe: protocolo, secretaria, tipo, valor estimado, data de autorização pelo Gabinete
And a lista é ordenada por data de entrada (FIFO)
```

### Cenário 2: Aprovar ordem para empenho
```gherkin
Given que analisei uma ordem com status "AGUARDANDO_CONTROLADORIA" e está em conformidade
When clico em "Aprovar" e confirmo
Then a ordem avança para status "AGUARDANDO_EMPENHO"
And o registro em ordem_historico contém: nome completo do fiscal, acao="aprovar"
And a secretaria e a contabilidade são notificadas (assíncrono)
```

### Cenário 3: Registrar irregularidade com descrição obrigatória
```gherkin
Given que identifiquei uma irregularidade fiscal ou legal na ordem
When clico em "Irregularidade" e preencho a descrição com pelo menos 50 caracteres
And confirmo a ação
Then a ordem muda para status "COM_IRREGULARIDADE" (suspensa)
And o registro em ordem_historico inclui a descrição completa do fiscal
And a secretaria é notificada (assíncrono)
And a ordem fica bloqueada até resolução manual pelo Administrador
```

### Cenário 4: Irregularidade sem descrição suficiente — bloqueado
```gherkin
Given que clico em "Irregularidade"
When submeto com descrição menor que 50 caracteres
Then o campo exibe: "A descrição da irregularidade deve ter no mínimo 50 caracteres (X/50)"
And nenhuma requisição é enviada ao back-end
```

### Cenário 5: Solicitar documentação adicional
```gherkin
Given que a ordem precisa de documentação adicional para análise
When clico em "Solicitar Documentação" e preencho a observação
Then a ordem muda para status "AGUARDANDO_DOCUMENTACAO"
And a secretaria é notificada sobre a solicitação
And quando a secretaria enviar os documentos, a ordem volta a "AGUARDANDO_CONTROLADORIA"
```

### Cenário 6: Ação em ordem com status incorreto é rejeitada
```gherkin
Given que uma ordem está com status "AGUARDANDO_GABINETE"
When o back-end recebe uma ação da Controladoria para essa ordem
Then recebo HTTP 422: "Transição inválida: status atual não permite esta ação"
```

### Cenário 7: Registro com nome completo do fiscal
```gherkin
Given que executo qualquer ação como fiscal da Controladoria
When a ação é concluída
Then o registro em ordem_historico contém o nome completo do usuário autenticado
And não apenas o ID ou e-mail
```

## Regras de Negócio

| ID | Regra |
|----|-------|
| RN-37 | Somente ordens com status AGUARDANDO_CONTROLADORIA ou AGUARDANDO_DOCUMENTACAO recebem ações desta tela |
| RN-38 | Parecer de irregularidade exige descrição obrigatória com mínimo 50 caracteres |
| RN-39 | Ordens com irregularidade ficam suspensas até resolução manual pelo Administrador ou nova aprovação |
| RN-40 | A Controladoria pode visualizar documentos anexados à ordem |
| RN-41 | Todo parecer é registrado com nome completo do fiscal responsável |

## Escopo

### IN ✅
- Tela de fila da Controladoria (status AGUARDANDO_CONTROLADORIA + AGUARDANDO_DOCUMENTACAO)
- Ações: aprovar → AGUARDANDO_EMPENHO, irregularidade → COM_IRREGULARIDADE, solicitar_documentacao → AGUARDANDO_DOCUMENTACAO
- Validação de 50 chars mínimos para irregularidade
- Registro em ordem_historico com nome_completo do fiscal
- Máquina de estados validando status permitidos

### OUT ❌
- Reversão de irregularidade (somente Admin)
- Upload de documentos pela Secretaria (essa ação é da secretaria, gerenciada via status)
- Visualização de documentos (mencionado em RN-40 mas sistema ainda não tem upload de arquivos implementado)
