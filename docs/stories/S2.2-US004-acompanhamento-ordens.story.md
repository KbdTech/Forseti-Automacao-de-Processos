# US-004 — Acompanhamento de Ordens pela Secretaria

**ID:** US-004
**Sprint:** S2 (Semanas 3–4)
**Status:** Ready
**Prioridade:** Alta
**Depende de:** US-003
**Estimativa:** 8 pontos
**Módulo:** Criação e Acompanhamento de Ordens

---

## Descrição

> Como **servidor de uma secretaria municipal**, quero visualizar todas as ordens da minha secretaria com seus status atuais e histórico de tramitação, para que eu possa acompanhar o progresso de cada demanda sem precisar contatar outros setores.

## Contexto e Motivação

Após a criação de uma ordem, o servidor precisa de visibilidade sobre o que aconteceu com ela. Esta US entrega a lista paginada de ordens da secretaria, com busca por protocolo, filtros por status, e o detalhe completo com histórico de tramitação.

---

## Critérios de Aceitação

### Cenário 1: Listagem paginada das ordens da própria secretaria
```gherkin
Given que estou autenticado com perfil "secretaria"
When acesso "Minhas Ordens"
Then vejo apenas as ordens da minha secretaria (filtro no back-end)
And a lista é paginada com 20 registros por página
And cada linha exibe: protocolo, tipo, status com badge colorido, data de criação, prioridade
And ordens de outras secretarias nunca aparecem
```

### Cenário 2: Busca por protocolo exato
```gherkin
Given que estou na tela "Minhas Ordens"
When digito o protocolo exato "OS-2026-00007" no campo de busca
Then o sistema filtra e exibe somente essa ordem
And buscas parciais como "00007" não retornam resultados (busca exata)
```

### Cenário 3: Filtro por status
```gherkin
Given que estou na tela "Minhas Ordens"
When seleciono o filtro de status "AGUARDANDO_GABINETE"
Then a lista exibe apenas as ordens com esse status
And o filtro pode ser combinado com a busca por protocolo
```

### Cenário 4: Debounce na busca
```gherkin
Given que estou digitando no campo de busca
When digito caracteres rapidamente
Then a requisição à API é disparada somente após 300ms de pausa na digitação
And não há requisições duplicadas durante a digitação
```

### Cenário 5: Visualização do detalhe completo com histórico
```gherkin
Given que estou na lista de ordens
When clico em uma ordem
Then um modal ou painel exibe todos os dados da ordem
And o histórico de tramitação é exibido em ordem cronológica crescente
And cada entrada do histórico mostra: data/hora, ação, perfil responsável, observação
```

### Cenário 6: Empty state para secretaria sem ordens
```gherkin
Given que sou um servidor de uma secretaria que ainda não criou ordens
When acesso "Minhas Ordens"
Then vejo uma tela de empty state com mensagem "Nenhuma ordem encontrada"
And um botão "Criar Nova Ordem" é exibido
```

### Cenário 7: Paginação funcional
```gherkin
Given que minha secretaria tem 45 ordens
When acesso "Minhas Ordens"
Then vejo as primeiras 20 ordens
And controles de paginação exibem "Página 1 de 3"
And posso navegar para páginas 2 e 3
```
