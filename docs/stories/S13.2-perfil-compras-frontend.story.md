# S13.2 — Frontend: Interface do Perfil `compras`

**ID:** S13.2
**Sprint:** S13
**Status:** Done
**Prioridade:** Alta
**Estimativa:** 3 pts
**Depende de:** S13.1 (enum `compras` disponível no backend)
**Módulo:** Frontend — Auth / Sidebar / Rotas / Admin

---

## Descrição

> Como **usuário com perfil `compras`**, após fazer login quero ser redirecionado diretamente para a tela de Gestão de Fornecedores e ver apenas esse menu no sidebar, sem visualizar pipelines ou áreas de outros setores.

## Contexto e Motivação

S13.1 adiciona o enum `compras` no backend. Esta story adapta o frontend para reconhecer o novo perfil em todos os pontos necessários:
- Tipo TypeScript `RoleEnum`
- Rota padrão pós-login
- Sidebar (item único: Gestão de Fornecedores)
- RoleGuard na rota `/admin/fornecedores`
- Dropdown de perfil na tela de Gestão de Usuários

O perfil `compras` tem a **tela mais restrita** do sistema: acessa apenas `/admin/fornecedores` com CRUD completo (herdando a tela já implementada em S11.2).

---

## Critérios de Aceitação

### Cenário 1: Redirecionamento pós-login
```gherkin
Given que faço login com credenciais de usuário com perfil 'compras'
Then sou redirecionado para '/admin/fornecedores'
And a tela de Gestão de Fornecedores carrega normalmente
```

### Cenário 2: Sidebar restrito
```gherkin
Given que estou logado com perfil 'compras'
When visualizo o sidebar
Then vejo apenas o item "Gestão de Fornecedores"
And não vejo nenhum pipeline, dashboard, auditoria ou gestão de usuários
```

### Cenário 3: Acesso direto a rotas proibidas
```gherkin
Given que estou logado com perfil 'compras'
When navego diretamente para '/gabinete'
Then sou redirecionado para '/acesso-negado'

When navego para '/secretaria/ordens'
Then sou redirecionado para '/acesso-negado'
```

### Cenário 4: Funcionalidades na tela de fornecedores
```gherkin
Given que estou logado com perfil 'compras'
When acesso '/admin/fornecedores'
Then posso criar, editar e ativar/desativar fornecedores
And a tela funciona idêntica ao que admin vê
```

### Cenário 5: Admin pode criar usuário com perfil 'compras'
```gherkin
Given que estou logado como admin
When acesso Gestão de Usuários e abro o modal de criação
Then o campo "Perfil" exibe a opção "Setor de Compras"
And posso criar um usuário com esse perfil
```

## Especificação Técnica

### 1. `frontend/src/types/auth.types.ts`

```typescript
// Antes
export type RoleEnum =
  | 'secretaria'
  | 'gabinete'
  | 'controladoria'
  | 'contabilidade'
  | 'tesouraria'
  | 'admin'

// Depois
export type RoleEnum =
  | 'secretaria'
  | 'gabinete'
  | 'controladoria'
  | 'contabilidade'
  | 'tesouraria'
  | 'compras'    // ← NOVO S13.2
  | 'admin'
```

### 2. `frontend/src/types/routes.ts`

**DEFAULT_ROUTE** — adicionar entrada:
```typescript
compras: '/admin/fornecedores',
```

**SIDEBAR_ITEMS** — adicionar `'compras'` no item existente:
```typescript
// Item já existente — apenas adicionar 'compras' ao array roles
{
  key: 'fornecedores-admin',
  label: 'Gestão de Fornecedores',
  path: '/admin/fornecedores',
  iconKey: 'Store',
  roles: ['admin', 'compras'],  // ← adicionar 'compras'
},
```

**ROLE_LABEL** — adicionar entrada:
```typescript
compras: 'Setor de Compras',
```

### 3. `frontend/src/routes/index.tsx`

Localizar a rota `/admin/fornecedores` e adicionar `'compras'` ao RoleGuard:

```tsx
// Antes
<RoleGuard roles={['admin']}>
  <FornecedoresAdminPage />
</RoleGuard>

// Depois
<RoleGuard roles={['admin', 'compras']}>
  <FornecedoresAdminPage />
</RoleGuard>
```

### 4. `frontend/src/pages/admin/UserManagementPage.tsx`

Localizar o `<Select>` ou `<SelectContent>` de perfis no modal de criação/edição de usuário e adicionar:

```tsx

```

> Verificar se o componente usa `ROLE_LABEL` dinamicamente ou tem items hardcoded.
> Se dinâmico, apenas o passo 2 (ROLE_LABEL) já resolve.
